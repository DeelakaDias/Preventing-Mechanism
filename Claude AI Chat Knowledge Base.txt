## 14. FEATURE ENGINEERING

### Feature Extraction
- 60 vibration bands: 10-20Hz to 600-610Hz
- 6 electrical features: VoltageMean, VoltageMin, VoltageMax, CurrentMean, CurrentMin, CurrentMax
- Power excluded — mathematically derived from voltage × current, would introduce redundancy
- **Total: 66 features per record**

### Frequency Group Splitting (FAG-TFT specific)
```
Group 1 (G1): indices 0-8   → 10-100Hz  (9 bands)
Group 2 (G2): indices 9-28  → 100-300Hz (20 bands)
Group 3 (G3): indices 29-59 → 300-610Hz (31 bands)
Group 4 (Elec): indices 60-65 → 6 electrical features
```

### Sequence Creation
- TIME_STEPS = 7
- Sliding window — each sequence contains 7 consecutive records
- Label = class of the 7th (last) record
- Creates temporal context for degradation detection

### Scaling
- StandardScaler fitted on training data only
- Applied to test data and live inference — never refit
- Prevents data leakage

### Train/Test Split
- 80% training, 20% testing
- Stratified split — preserves class proportions in both sets
- random_state=42 for reproducibility

## 18. MODEL 3 — CNN-LSTM DUAL BRANCH

**Architecture:**
```
Input (32, 7, 66)
    ↓ split
Vibration branch (60 features per timestep):
    Conv1d(1→64, k=3) + BN + ReLU
    Conv1d(64→32, k=3) + BN + ReLU
    MaxPool1d(2) + Dropout
    Flatten → 960 features per timestep

Electrical branch (6 features per timestep):
    Linear(6→16) + BN + ReLU

Merge: (960+16) = 976 per timestep
    ↓
LSTM(976→64, 2 layers, dropout=0.3)
    ↓ last timestep
Dense(64→32) + ReLU
    ↓
Dense(32→5)
```

**Why dual branch:**
- Conv1D needs a sequence dimension — 60 vibration bands treated as 1D signal
- Electrical features have no neighbourhood relationship — Dense is appropriate
- Prevents Conv1D from finding spurious patterns between vibration and electrical features

**Results:**
```
Accuracy : 92.04%
F1 Score : 0.8823
RMSE     : 0.5439
MAE      : 0.1386
Breakdown detection: 0.00 for all breakdown classes — collapsed to Healthy-only
```

Despite class weights — 309,045 parameters too complex for 85 breakdown samples per class.

## 19. MODEL 4 — FAG-TFT (Novel Architecture)

### Full Name
**Frequency-Aware Gated Temporal Fusion Transformer**

### Two Novel Contributions

**Contribution 1 — Hierarchical Variable Selection Network**

Standard TFT VSN treats all features equally. FAG-TFT groups vibration bands by physical significance before selection:

- Within-group: GRN processes each group independently
- Within-group gating: sigmoid weights decide which features within each group matter
- Across-group: softmax weights decide which GROUP matters most
- Produces interpretable group importance weights per prediction

**Contribution 2 — CNN-LSTM Autoencoder Gating Mechanism**

- Trained on healthy data only
- Acts as anomaly detector before classifier runs
- Low error → healthy → skip classifier
- High error → anomaly → activate classifier
- Handles States 4 and 5 (unknown faults)

### FAG-TFT Internal Components

**GRN (Gated Residual Network):**
```
input → Linear → ELU → Dropout → gate(sigmoid) × Linear → LayerNorm(output + residual)
```
Gates irrelevant information to zero. Residual connection preserves original information. Core building block used throughout FAG-TFT.

**HierarchicalVSN:**
```
Per timestep:
G1(9) → GRN → 64 → sigmoid gate → weighted h1
G2(20) → GRN → 64 → sigmoid gate → weighted h2
G3(31) → GRN → 64 → sigmoid gate → weighted h3
Elec(6) → GRN → 64 → sigmoid gate → weighted he
[h1,h2,h3,he] concat → GRN → softmax → 4 group weights
Output: group_merge(concat) + weighted_sum → 64
```

**FAG_TFT Classifier forward pass:**
```
For each of 7 timesteps:
    Split into G1, G2, G3, Elec
    → HierarchicalVSN → 64 features
Stack 7 outputs → (batch, 7, 64)
    → LSTM encoder (2 layers) → (batch, 7, 64)
    → Multi-Head Attention (4 heads) → (batch, 7, 64)
    → LayerNorm with residual
    → take last timestep → (batch, 64)
    → GRN post-attention
    → Dense(64→32) + ReLU + Dropout
    → Dense(32→5)
```

**AnomalyGate (CNN-LSTM Autoencoder):**
```
ENCODER:
Per timestep: Conv1d(1→32) + BN + ReLU + MaxPool → flatten → 960
Merge with elec → 966
LSTM(966→32) → last hidden state → Linear(32→16) [bottleneck]

DECODER:
Repeat bottleneck 7 times → (batch, 7, 16)
LSTM(16→32) → Linear(32→66) → reconstructed (batch, 7, 66)

Loss: MSE between input and reconstruction
```

## 21. TRAINING CONCEPTS

**Batch Size: 32**
- 4292 training sequences ÷ 32 = 134 batches per epoch
- Model updates weights 134 times per epoch
- DataLoader shuffles sequences each epoch

**Epoch**
- One complete pass through all training data
- 150 epochs maximum with early stopping

**Class Weights**
```
class_counts  = [85, 3951, 87, 85, 84]
class_weights = [1.247, 0.027, 1.218, 1.247, 1.262]
```
Breakdown classes penalised ~47× more than Healthy for misclassification.

**Early Stopping**
- Monitors validation loss
- Saves best model weights when validation loss improves
- Stops training when validation loss does not improve for PATIENCE epochs

**Gradient Clipping: 1.0**
- Prevents exploding gradients in LSTM/recurrent networks
- Caps maximum gradient norm at 1.0

**StandardScaler**
- Fitted on training data only — never refit on test or live data
- Transforms each feature to mean=0, std=1
- Prevents data leakage

**Dropout: 0.3**
- Randomly disables 30% of neurons during training
- Disabled during inference
- Reduces overfitting

**Weight Decay: 1e-3**
- L2 regularisation — penalises large weights
- Forces model to generalise rather than memorise

## 26. KNOWN PROBLEMS AND LIMITATIONS

### Class Imbalance (47:1)
All four classifiers defaulted to predicting Healthy for everything. Fixed with class weighting but breakdown detection still poor due to insufficient real breakdown data.

### Anomaly Gate Not Fully Converged (First Run)
Initial training stopped at 100 epochs — still learning. Fixed by increasing to 300 epochs with patience=30.

### False Positives on Different Fabric Types
Machine sewing panel fabric caused high reconstruction error flagged as anomaly — but machine was healthy. Causes:
- Limited training data variety (one fabric type)
- No fabric type feature in model
- Thresholds too tight
- Bottleneck (16) too small for natural variation

### Small Dataset
Only 7 real breakdown records across 4 breakdown types. Synthetic data calibrated from these 7 records compensates partially but cannot replace diverse real observations.

### Cold Start
First 7 records of any machine use padding (earliest record repeated). Slightly reduces prediction confidence for initial strokes.

### CNN-LSTM TimeDistributed Error (Fixed)
Original architecture used TimeDistributed(Conv1D) on flat 66-feature vector — Conv1D requires 2D input (steps, channels). Fixed by dual-branch design separating vibration (reshaped to 60×1) from electrical (Dense).

### Overfitting
All models show train loss decreasing while val loss increases from early epochs. Addressed with dropout=0.3, weight_decay=1e-3, lower learning rate.

## 28. EVALUATION METRICS

| Metric | What It Measures |
|---|---|
| Accuracy | % of all predictions correct |
| F1 Score (weighted) | Harmonic mean of precision and recall, weighted by class support |
| RMSE | Root mean squared error — treats labels as numbers |
| MAE | Mean absolute error — average distance from true label |
| Precision | Of all times model predicted class X, how many were correct |
| Recall | Of all actual class X samples, how many did model catch |
| Confusion Matrix | Per-class breakdown of correct and incorrect predictions |
